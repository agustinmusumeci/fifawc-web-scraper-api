# Contributing to the API ❤️

Thanks for thinking about contribute!

Please read the relevant section before making your contribution. It will make it a lot easier for the mainteiner [agustinmusumeci][author_url] and smooth out the experience for all involved.

All types of contributions are encouraged and valued.

## Send feature requests using GitHub's Pull Requests

We actively welcome you to [pull request][gh_pull_request_url]:

1. Fork the repo and create your own branch from `main`;
2. If you've added code that should be tested, add tests;
3. If you've changed project's APIs, update the documentation;
4. Make sure that your code has successfully passed the linters;
5. Make sure that your tests were successful (we will be adding test soon).

> I recommend you to create a new issue that covers information about your pull request.

## Report bugs using GitHub's Issues

Report a bug by [opening a new issue][gh_issues_url].
